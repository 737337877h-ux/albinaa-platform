# Release Notes — Albinaa Platform v0.1 Alpha

**تاريخ الإصدار:** 22 يوليو 2026
**نوع الإصدار:** Alpha — إصدار تجريبي أولي
**رقم الإصدار:** v0.1.0-alpha

---

## ملخص

إصدار أولي لمنصة **Albinaa** لإدارة وتحصيل الديون. يوفر النظام آليات متكاملة لإدارة العملاء، استيراد البيانات المالية، تتبع المتابعة، إدارة الوعود، وتسجيل عمليات التحصيل.

---

## الميزات المنجزة

### 1. نظام إدارة المستخدمين والصلاحيات
- تسجيل دخول بالاسم وكلمة المرور مع JWT
- 4 أدوار:مدير (admin)، محصل (collector)، مراجع (reviewer)، عارض (viewer)
- صلاحيات دقيقة لكل دور (إنشاء/عرض/تعديل/حذف)
- تغيير كلمة المرور الإجباري عند أول دخول (`mustChangePassword`)
- تسجيل خروج مع إبطال Refresh Token
- Rate Limiting على نقاط الدخول الحساسة (5 محاولات/دقيقة)

### 2. إدارة العملاء (Customer 360)
- قائمة عملاء مع البحث والفلترة حسب الفرع والعملة
- ملف شامل لكل عميل يشمل:
  - البيانات الأساسية (الاسم، الهاتف، البريد)
  - الأرصدة حسب العملة (SAR / YER)
  - سجل العمليات (Timeline)
  - كشف حساب بسيط
  - المتابعة والوعود
- تحديث تلقائي للأرصدة بعد كل عملية

### 3. استيراد البيانات (Import)
- رفع ملفات Excel (.xlsx)
- تحليل مبدئي (dry_run) لمعاينة النتائج قبل التنفيذ
- قواعد تحقق صارمة:
  - منع التكرار بـ `line_hash` الفريد
  - حماية البيانات المُلّاة بالفعل (الحذف يُمنع)
  - دعم العملات المتعددة (SAR, YER)
- تقرير استيراد تفصيلي (عدد الصفوف، العملاء الجدد، التكرارات، الأرصدة)
- إعادة رفع نفس الملف = خطأ 409 (عدم التكرار)

### 4. دورة التحصيل (Collection Cycle)
- تسجيل التحصيل مع:
  - تحديد العملة والمبلغ (سالب = دين جديد، موجب = تحصيل)
  - طريقة الدفع (نقدي/تحويل/شيك/محفظة)
  - رقم المرجع واسم البنك
  - ملاحظات
- عكس التحصيل (Reversal) مع تأكيد إضافي
- تحديث الأرصدة تلقائيًا بعد التحصيل أو عكسه

### 5. إدارة الوعود (Promises)
- إنشاء وعد جديد (upcoming)
- تعديل تاريخ الوعيد (upcoming فقط)
- تنفيذ الوعيد → `fulfilled`
- حذف الوعيد المُلغى → `deleted`
- رفض تعديل الوعيد المُنفَّذ (409)
- جدول مهام متزامن مع الوعود

### 6. نظام المتابعة (Follow-ups)
- تسجيل متابعة مع نوع (هاتف/رسالة/زيارة/بريد)
- تقييم المتابعة (نجاح/جزئي/فشل/لم يجب)
- تحديد تاريخ المتابعة التالي
- جدولة المهام تلقائيًا

### 7. إدارة المهام (Tasks)
- عرض المهام بناءً على الدور:
  - المحصل: مهامه فقط مع تبويبات (اليوم/القادمة/المتأخرة/الكل)
  - المدير/المراجع: مهام فارغة (لا مهام شخصية)
- تحديث حالة المهام (مكتمل/ملغي)

### 8. إعدادات النظام (Branches / Roles / Users)
- إدارة الفروع (إنشاء/عرض)
- إدارة الأدوار (إنشاء/عرض مع الصلاحيات)
- إدارة المستخدمين:
  - إنشاء مستخدم جديد مع كلمة مرور مؤقتة
  - تعطيل/تفعيل المستخدمين
  - إعادة تعيين كلمة المرور

### 9. واجهة عربية/RTL بالكامل
- واجهة عربية 100% (RTL)
- أرقام عربية عند العرض
- خطوط عربية مدعومة (IBM Plex Arabic)

### 10. الأمان
- JWT مع Access Token (15 دقيقة) و Refresh Token (7 أيام)
- Argon2id لتشفير كلمات المرور
- حماية الجداول Append-Only
- تسجيل Audit Log لكل العمليات
- Rate Limiting على تسجيل الدخول

## القيود الحالية

1. **بيئة Alpha**: غير مناسب للإنتاج — بيانات تجريبية فقط
2. **قاعدة واحدة**: لا يوجد Multi-Organization بعد
3. **عملات محدودة**: SAR و YER فقط (قابل للتوسيع من الإعدادات)
4. **لا يوجد PWA**: لا يدعم التثبيت على الجوال
5. **لا يوجد Offline**: لا يعمل بدون اتصال بالإنترنت
6. **لا يوجد CI/CD**: لا يوجد خط أنابيب آلي
7. **لا يوجد مراقبة**: لا يوجد Prometheus/Grafana
8. **لا يوجد ترجمة**: عربية فقط (قابل للتوسيع)
9. **ملف Excel واحد فقط**: لا يوجد دعم CSV أو ربط API خارجي
